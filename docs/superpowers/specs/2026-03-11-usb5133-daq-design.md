# NI USB-5133 데이터 수집 시스템 설계 문서

**날짜:** 2026-03-11
**상태:** 승인됨

---

## 개요

NI USB-5133 디지타이저를 활용한 Python 기반 데이터 수집 및 처리 시스템.
실시간 파형 모니터링, FFT 신호 분석, CSV 저장 기능을 PyQt5 GUI로 제공한다.

---

## 요구사항

- **언어:** Python 3.x
- **디바이스:** NI USB-5133 (2채널, 8비트, 최대 100 MS/s)
- **기능:**
  - 실시간 파형 모니터링 (1~2채널 유연하게)
  - FFT 스펙트럼 분석
  - CSV 파일 저장
- **UI:** PyQt5 + pyqtgraph GUI
- **저장 형식:** CSV

```
├── ui/

```

### 데이터 흐름

```
NI USB-5133
    ↓  (niscope API)
device/scope.py
    ↓  (numpy array)
acquisition/worker.py  (QThread)
    ↓  (Qt Signal: data_ready)
    ├→ ui/waveform_plot.py    (실시간 파형 표시)
    ├→ analysis/fft.py        (FFT 계산 → fft_plot.py)
    └→ storage/csv_writer.py  (사용자 저장 버튼 클릭 시)
```

## 컴포넌트 상세

### `device/scope.py`

- **`NI5133` 클래스**
  - `connect()` — NI-SCOPE 세션 열기. **`configure()` 전에 반드시 호출해야 함**
  - `disconnect()` — 세션 닫기
  - `configure(sample_rate, record_length, channels, voltage_range)` — 수집 파라미터 설정
    - `sample_rate`: Hz 단위 float (최대 100,000,000 Hz). 초과 시 예외 발생
    - `record_length`: 정수, 단위는 **샘플 수** (예: 1000 → 샘플레이트 1 MS/s 기준 1 ms 수집)
    - `voltage_range`: ±0.5 V / ±1 V / ±2 V / ±5 V 중 하나
  - context manager 지원 (`with NI5133() as scope:`) — `__enter__`에서 `connect()`, `__exit__`에서 `disconnect()` 호출
- **`MockScope` 클래스** — 하드웨어 없는 환경에서 테스트용 사인파 생성, `NI5133`과 동일한 인터페이스

### `acquisition/worker.py`

- `QThread` 서브클래스
- **Signals:**
  - `data_ready = pyqtSignal(object)` — `np.ndarray` shape `(num_channels, record_length)` 전달
    - `waveform[0]`: CH0 데이터, `waveform[1]`: CH1 데이터 (1채널 모드 시 shape `(1, record_length)`)
  - `error_occurred = pyqtSignal(str)` — 오류 메시지 전달
- **메서드:**
  - `stop()` — 스레드 안전 플래그(`threading.Event`) 설정 후 run 루프 종료, NI-SCOPE fetch 완료 후 안전하게 종료
- 수집 모드: 연속(continuous) / 단발(single-shot)
- 수집 파라미터: 샘플레이트, 레코드 길이, 활성 채널

### `analysis/fft.py`

- `compute_fft(waveform: np.ndarray, sample_rate: float) -> (freqs, magnitudes)`
- numpy.fft 기반, Hanning 윈도우 적용
- 단측 스펙트럼 반환

### `storage/csv_writer.py`

- `save_waveform(waveform: np.ndarray, sample_rate: float, filename: str)`
- `waveform` shape: `(num_channels, record_length)`
- 시간축 자동 생성 후 CSV 저장, 컬럼: `time, ch0[, ch1]`
- `numpy.savetxt` 기반 (pandas 미사용)

### `ui/main_window.py`

- **상단 컨트롤 패널:**
  - 연결/해제 버튼, 채널 선택 체크박스 (CH0 / CH1)
  - 샘플레이트 입력 (유효 범위: 1 ~ 100,000,000 Hz, 초과 시 UI에서 경고)
  - 전압 범위 콤보박스 (±0.5 V / ±1 V / ±2 V / ±5 V)
- **중앙 플롯 영역:** 파형 플롯 (상) + FFT 스펙트럼 플롯 (하)
- **하단 패널:** 시작/정지 버튼, CSV 저장 버튼, 상태바
- **마지막 파형 캐시:** `data_ready` Signal 수신 시 `main_window`가 마지막 `np.ndarray`를 내부에 보관. CSV 저장 버튼 클릭 시 이 캐시를 `csv_writer`에 전달

## 에러 처리

| 상황 | 처리 방법 |
|------|-----------|
| 디바이스 미연결 | GUI 오류 다이얼로그 표시, 앱 계속 실행 |
| 수집 중 오류 | `error_occurred` Signal → 상태바 표시, 수집 자동 정지 |
| 하드웨어 없는 환경 | `MockScope`로 대체하여 개발/테스트 |
